Dark mode is one class: .sg-dark re-binds the same token names (pearl→#141016, ink→#F4EEF1, glass darkened). No component CSS changes — everything reads tokens.
One data field to add (flag). The swatch smear is data-driven: each item needs a colour triplet --s1 / --s2 / --sdeep (light → mid → deep). The Sheet has no hex column yet — add one, or derive at entry/scan time. Card legibility is solved by the fixed scrim, so any shade is safe.

03 Interaction & motion spec

One line per micro-animation, with its reduced-motion fallback. Two easings only: --sg-spring (cubic-bezier .34,1.56,.64,1) for tactile presses & entries, --sg-ease (.22,.61,.36,1) for everything else.

  • Gloss sweep.pc--active .pc__smear::before
    A diagonal white streak crosses the active swatch every 4.6s — the "dewy" tap shimmer. The signature moment.reduced-motion — animation removed; card shows its static end state (no streak).
  • Screen change.screen
    New tab fades + rises 8px over .26s ease; outgoing fades out.reduced-motion — opacity-only cross-fade, no translate.
  • Tap press.row / .btn / .tab--scan :active
    Scale .975–.94 for ~.15s spring — physical "give" on press.reduced-motion — drop scale; keep a brief background flash for feedback.
  • Row reveal.row.revealed .row__actions
    Long-press (420ms) slides Opened / Retire in from the right, .26s spring; row content dims to .25.reduced-motion — actions appear instantly, no slide.
  • Bottom sheet.sheet + .scrim
    Duplicate sheet springs up from the bottom .34s; scrim blur-fades .26s. Fires on Save, never mid-typing.reduced-motion — sheet appears in place; scrim fades opacity only.
  • Toast.toast
    Rises 14px + fades in .3s spring, auto-dismiss 2.6s. Carries an optional inline action.reduced-motion — fade only; same timing.
  • Scan read.spin → result
    Shutter → 1.3s spinner ("Reading barcode…") → hit or miss card swaps in; cursor auto-lands on the shade field.reduced-motion — replace spinner rotation with a static "Reading…" label + pulse.
  • Search filter#list-rows
    Results re-render on every keystroke after ~2 chars — no submit. Match can surface above the keyboard.reduced-motion — unaffected (no motion).
  • Haul subtotal.haul-item.done
    Check toggles → item dims + strikes through; sticky ₩ total recomputes live (spent ↑, left ↓).reduced-motion — instant state change.
  • Mode switch.app / .screen
    Light ↔ dark cross-fades background & text .35s ease so the flip isn't jarring.reduced-motion — instant swap.
  • Caret blink.caret
    Search caret blinks 1.1s when active — life signal.reduced-motion — solid caret, no blink.
  • Money-path playplayMoney()
    Scripted demo: open → type "pure cake" (~95ms/char) → owned card → scan → save, with iridescent coachmarks.reduced-motion — keep, but skip the per-character typing (set the value at once).
Global guard. Wrap decorative motion in @media (prefers-reduced-motion: no-preference) (as the gloss sweep already is) so print, PDF export and reduced-motion users always get the legible end state, never a pre-animation opacity:0.

Touch & accessibility floor

  • Hit targets
    ≥44px everywhere; primary actions sit in the thumb zone (bottom 1/3). Shutter 70px, tab items 58×46.
  • Focus
    Visible accent ring on inputs/search (0 0 0 2px rgba(255,77,98,.3)); never remove outlines without a replacement.
  • Long strings
    Shade code is flex:none — it never truncates. Names ellipsize instead. The code is identity.
