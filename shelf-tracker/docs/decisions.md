# decisions.md

Dated log of architectural/UX decisions. Claude Code appends here per CLAUDE.md.

- 2026-06-11 — v0 scaffolded in Claude.ai chat: GAS tracker + Claude-vision scanner + static catalog. Sheet = source of truth. Seoul Glass direction defined; Claude Design bundle (docs/design/) supersedes CLAUDE.md tokens when present.

- 2026-06-13 — **Seoul Glass implementation (P1–P6 in one pass).** Rebuilt `apps-script/index.html` from the v0 utilitarian shell into the full Seoul Glass design from the Claude Design handoff bundle (`design_handoff_shelf_tracker/`), wired to the live backend. Scope was confirmed as "full app + extend backend, Tracker only."
  - **Tokens adopted** verbatim from `hifi/seoul-glass.css`: pearl `#F7F4F5`/ink `#1A1518`, glass `rgba(255,255,255,.72)` + 20px blur + inner highlight, accent tint-red `#FF4D62`, iridescent `#9DB8FF→#FFB7C9` (hero/active only), status fresh `#3E7D5C` / soon `#A85F12` / expired `#B8323E`, radii 20/16/999, springs `cubic-bezier(.34,1.56,.64,1)` / `(.22,.61,.36,1)`, Pretendard + Spline Sans Mono with `PingFang TC`/`Noto Sans TC` fallbacks. Dark mode via `.sg-dark` (system default + manual moon/sun toggle, persisted to `localStorage.st.dark`).
  - **Surfaces shipped:** List (search-first, filter chips, photocard rows, long-press → Opened/Retire, tap → Detail), Add (brand autocomplete from her own history + duplicate guard), Scan (viewfinder → BarcodeDetector/html5-qrcode → OBF lookup → hit/miss; miss is neutral, never error styling), Import (paste preview with client-side dupe-skip), Catalog (Grid photocards / Insights computed from bootstrap / Korea Haul), Detail (swatch hero + "you already own this" ribbon). Tab bar List·Add·Scan(raised)·Import·Catalog.
  - **P3 duplicate guard:** client-side on Add-save and per Import row — brand+shade_number (case-insensitive, trimmed), falling back to brand+product+shade_name when no code. Bottom sheet "You own this — added <date> · Cancel / Save anyway."
  - **Backend extensions (Code.gs):**
    - Added 16th column `swatch_color` (hex). `ensureSchema()` now guards `getMaxColumns()` before writing headers; `getBootstrap()` returns `color` + `price`/`pd`/`img`/`notes` per row; `addItem`/`bulkImport` persist it (back-compat: 15-col scanner TSV pads to 16). Empty colour falls back to a deterministic client-side triplet from a curated K-beauty palette (handoff flagged this field; "any colour is safe — scrim guarantees legibility").
    - New `haul` sheet tab (created on first use) + `getHaul` / `addHaulItem` / `setHaulChecked` / `removeHaulItem`. String-safe writes (shade_number text, never `appendRow`). Korea Haul check-state now persists across devices with a live ₩ subtotal. (Supersedes the P2 plan to keep haul in localStorage.)
  - **Effective-expiry rule unchanged** (earliest of printed / opened+PAO / mfg+36mo "est."); presentation only — fresh/soon(≤90d)/expired/`no date` badges, plus a `none` tone for undated items.
  - **Verification harness:** `tools/build-preview.js` strips the GAS scriptlet (→`"{}"`), tags `<body class="preview">` (device-frame CSS), and injects `window.__MOCK__` seeded from `data/inventory-snapshot-2026-06-11.json` so every `google.script.run` call resolves in a plain browser. The shipped `index.html` is never modified; it uses a `RUN()` bridge that prefers real `google.script.run` and falls back to the mock. Generated `apps-script/preview.html` (gitignored) is for QA only.
  - **Verified in-browser (all 201 real items + 6 wishlist):** every screen light + dark, search filter, expiry sort, dupe sheet, scan miss, import dupe-skip count, catalog grid/insights/haul live subtotal, detail hero, long-press actions. `new Function()` parses the single inline script block; `<base target="_top">` + scriptlet preserved; 44px targets; reduced-motion kill-switch.
  - **Not touched this session:** `scanner/shelf-scanner.html` (Seoul Glass reskin deferred), `makeup-shelf-check.html` (now superseded by the Catalog tab — keep until parity confirmed, then deprecate per README).

- 2026-06-13 (follow-up) — **GitHub backing + clasp + Scanner reskin + P5.**
  - **Git:** repo initialized on `main`, pushed to private **github.com/zmzhong1/shelf-tracker**. Follow-up work on branch `seoul-glass-followups` → PR.
  - **Clasp scaffold:** `package.json` (clasp + preview scripts), `apps-script/appsscript.json` (V8 manifest, web app *execute as Me / access Only myself*), `.clasp.json.example`. `build-preview.js` now writes `preview.html` to the repo root (not `apps-script/`) so clasp `rootDir: apps-script` pushes only Code.gs + index.html + appsscript.json. Final `clasp login` + scriptId are user-supplied (can't be done headlessly). Documented in `SETUP.md`.
  - **P7 multi-user template:** `SETUP.md` covers first-time manual deploy, clasp push-to-deploy, and the multi-user path (a blank Google Sheet *is* the template — `ensureSchema` builds 16 cols + the `haul` tab on first run).
  - **Scanner reskin (`scanner/shelf-scanner.html`):** ported to Seoul Glass (window-chrome panel, photo queue with read/reading/fail states + real thumbnails, amber "review before trusting" banner, glass results table, confidence pills, low-confidence cells tinted for review with edit-to-clear). Logic preserved exactly: keyless `api.anthropic.com` call, model `claude-sonnet-4-20250514`, `max_tokens:1000`, JSON parse, **16-col Import TSV** (was 15; col 16 swatch_color blank → derived on import). Added a `confidence` field to the vision prompt. Dark mode via `prefers-color-scheme`, **no localStorage** (hard rule kept).
  - **P5 photocard export:** Detail hero gets a download button → `drawCard()` renders the swatch card (smear gradients + scrim + brand/badge/name/code/shade) to a canvas with **no libraries** (Canvas 2D only, fonts via `document.fonts.load`), `toDataURL('image/png')` → download. Code font auto-fits width; canvas is untainted (all generated, no external images) so export always works.
  - **Verified in browser:** scanner light+dark with mock rows (confidence tinting, edit-clears-flag, Chinese render); photocard PNG render. All inline scripts pass `new Function()`.
  - **Still open:** none of the original backlog — P1–P7 all addressed. `makeup-shelf-check.html` still pending formal deprecation once Catalog parity is confirmed in daily use.
